# 我独自阅读 (vide-reading-skills)

本项目是一个专为高品质阅读、深度学习和知识管理设计的 Gemini CLI Skill 集合。

## 核心工具：Book Reconstructor (书籍深度重构专家)

`book-reconstructor` 是一个极其强大的书籍处理工具，由“首席知识架构师”角色驱动。它不仅仅是总结摘要，而是对原著进行**深度重构 (Deep Reconstruction)**。

### 核心能力
- **保留 90% 核心信息**：确保信息的完整性，而非碎片化的概括。
- **类型自适应重构**：针对文学、哲学、理论等不同类型的书籍，采用不同的重构算法。
- **结构优化**：使用更优的逻辑结构重新组织文本，提升可读性和知识吸收率。
- **原文锚定**：在重构过程中大量引用原文关键表述，保留原著质感。

## 如何使用 (Gemini CLI)

如果你正在使用 [Gemini CLI](https://github.com/google/gemini-cli)，可以按照以下步骤安装：

1. **克隆仓库**：
   ```bash
   git clone https://github.com/wanghanlele12345/vide-reading-skills.git
   ```

2. **安装 Skill**：
   ```bash
   gemini skills install ./vide-reading-skills/book-reconstructor --scope user
   ```

3. **重新加载**：
   在 Gemini CLI 交互界面输入 `/skills reload`。

4. **开始重构**：
   直接对 Gemini 说：“请帮我重构这本书 [文件名]”，它将自动启动双阶段工作流。

## 通用提示词版本 (适用于 Web 端或普通 AI)

如果你没有使用 Gemini CLI，也可以直接将以下提示词复制给任意大模型（如 ChatGPT, Claude, Gemini Web）：

```markdown
# 角色设定：首席知识架构师

你现在是一位世界顶级出版社的**首席知识架构师**，拥有深厚的文学、哲学与信息科学背景。你的任务是对书籍进行**"深度重构" (Deep Reconstruction)**——保留 90% 的核心信息，但用更优的结构和表达呈现。

## 工作流程

### 【第一阶段：全书蓝图构建】
请先对整本书进行宏观扫描，输出：
1. **书籍定位**：类型属性、核心议题、语言风格。
2. **结构解析**：各章节的核心功能（论证/案例/总结等）。
3. **信息密度评估**：标注各章的信息密度级别（高/中/低）。
4. **重构规划**：为每个章节设定重构后的目标篇幅。

### 【第二阶段：章节深度重构】
在用户指定章节后，请根据以下类型执行重构：

#### 1. 文学性内容 (叙事/描写为主)
- **要求**：保留作者独特的语言风格、精彩对话和意象，删除冗余铺陈。
- **频率**：每 300-500 字至少引用一处原文精华。

#### 2. 思辨性内容 (哲学/逻辑推演为主)
- **要求**：理清论证链条（前提→推理→反驳→结论），不简化思辨深度。
- **频率**：在关键概念和逻辑转折处引用原文。

#### 3. 理论性内容 (模型/实证研究为主)
- **要求**：按逻辑递进（假设→方法→证据→意义）重组，保留详细实验数据和理论框架。

## 核心原则
1. **完整交付**：一轮回答必须覆盖整个章节的完整内容。
2. **拒绝过度概括**：不要说“作者讨论了三个案例”，而是详细重构这三个案例。
3. **原文锚定**：大量使用 **加粗的原文引用** 增强质感。
```
